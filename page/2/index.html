IDL对于遥感影像的处理使用是非常方便的，这里主要总结一些使用技巧方面的。

nc文件的读取
PRO nc_open
;定义文件路径
MyRootDir=’E:\Documents\Desktop\’
MyRootDir2=’F:\huang\课程\海洋数值模拟\Aqua\CHL\2015\’

filearr = file_search(MyRootDir,’*.nc’,count=num);
FOR fileindex=0,num-1,1 DO BEGIN

nid = ncdf_open(filearr[fileindex], /nowrite )
; inquire about this file; returns structure
file_info = ncdf_inquire( nid )
; print out the dimensions of this file
FOR dimid=0, file_info.ndims -1 DO BEGIN
ncdf_diminq, nid, dimid, name, size
print, ‘ —> dimension ‘ + name + ‘ is: ‘, size
ENDFOR

FOR varid=0, file_info.nvars-1 DO BEGIN

ENDFOR

FOR num=0,(size(parentId))(1) -1 DO BEGIN
  parentName = NCDF_GROUPNAME(parentId[num])
  print, string(num) +' ---&gt; parent group Name ' + parentName

一个nc文件的主要框架是部分变量+Group数据，有时候数据保存在变量里（这也是大部分的nc文件思路），一般使用ncdf_varid就解决了（这样的解决思路网络上都有）。

nid = ncdf_open(filearr[fileindex], /nowrite )
chlid = ncdf_varid(nid, ‘chlor_a’)
但还有时候会保存在Group里面，通常我们可以发现文件大小挺大，但是利用ncdf_varid就是找不到数据，这时候需要看文件的数据是不是保存在了Group里面了。通过查询IDL官方文档，我发现了NCDF_GROUPSINQ，NCDF_GROUPNAME这两个函数，例子如下。

parentId = NCDF_GROUPSINQ(nid)

颜色表
通常我们会利用IDL进行作图，这样就离不开使用IDL自带的颜色表。IDL颜色表不仅多，而且可以自己定制，大家只要学会如何操作即可。
我主要讲一点，IDL颜色表的操作要优于matlab，我通过对比发现，matlab颜色表不仅少，而且不会自动调节，只会根据最大，最小值，然后间断颜色，不像IDL会根据数据分布来进行颜色表区分，使得数据的区域分布显现的更好，所以如果颜色表作图，更推荐IDL。

永恒之蓝总结分析

发表于 2017-05-17

受影响组织及初步影响

消极影响：主要是教育网的用户，英国医院，政府组织等等，凡是使用了老版微软系统的用户，基本都受到了冲击。给这些组织都带来了很大的经济损失，给组织里面的个人敲响了信息安全的警钟。互联网已经很久没有出现如此大的安全问题了。这是一次系统方面出现的漏洞，是偏底层的。越是底层的漏洞，其危害就越大。正如这次，就算你装了安全软件，但是你只要端口没有封，那么你就会被攻击成功。

积极影响： 重新敲响了个人用户互联网安全的意识，让大众重新认识到了信息安全的重要性。让每个人都养成良好的信息保密习惯，例如及时备份，及时安装补丁，重视信息安全。

经济影响

损失：被攻击的个人和组织都受到不同程度的损失，以及被攻击系统停机状态下所造成的其他损失。

获利：攻击者通过勒索病毒获取经济利益，以及附带的隐形效益，例如，全球安全公司的股票上涨，黑客组织可以利用卖工具和攻击的数据赚钱（影子经纪人）。也就是凡是保护数据安全的以及可能造成的数据不安全的团队都会因此而获利。这是攻防双方的获利，这是产业的红利。至于其他对个人的影响，或许就是信息安全专业更加好找工作，以及信息安全观念将成为人们的普遍意识。

政治影响

政治是各方利益妥协的结果，但是现在出现了一个利器，只需要投入人力以及计算机，就可以造成大规模的破坏。这样的武器怎么会不被掌握在暴力机构手里面呢？估计从此以后，政府攻击成为常态，而且系统安全性将会受到无比的重视，如何开发新一代系统成为每一个追求信息安全国家头上的问题。一个系统从诞生到使用，再到产业的繁荣，要很多年。做起来的确很辛苦，但是必须要做。

事件分析

这次永恒之蓝利用的漏洞在一个月前就已经被公布出来，当时的确在安全界引起一阵浪潮，但是没有引起足够的重视，以及没有对此漏洞可能作出的影响加以分析。例如：如果360公司在漏洞出现后，随后给所有安装360安全卫士的电脑打上微软提供的补丁，那么在这次被攻击的电脑中所有装了360安全卫士的电脑都幸免于难，那么这是多么好的公关和企业信誉的体现啊！这的确是所有安全公司决策者的失误。
一个漏洞出现了，一个月之后，蠕虫攻击病毒出现了，这场攻击是准备已久的，那么必然，攻击的只有在攻击者满足了和他无法再攻击之后才能停止。因此，随之而来的，是病毒的变种和增加。

二代病毒：继续扩散，利用已中毒电脑（此处应该利用好已中毒电脑，例如：已中毒电脑修补漏洞无用，或者已中毒电脑只要联网，就能被攻击）
后代病毒：利用好前辈占领的阵地，从而扩大影响。

《统计学习方法》第一章习题

1.1

三要素：模型，策略，算法

模型：伯努利模型是定义在取值为0与1的随机变量上的概率分布
策略：对数损失函数 L(Y,P(Y|X)) = -logP(Y|X)
算法：极大似然估计所使用的算法是求取经验风险函数的极小值，贝叶斯估计所使用的算法是求取参数的后验分布，然后计算其期望（贝叶斯不太懂）

为了得到使经验风险最小的参数值，使用的算法都是对经验风险求导，使导数为0

极大似然估计

经典统计学派：模型已定，参数未知，参数是固定的，未知
P(A=1)=θ,P(A=0)=1−θ
独立抽取n个同分布的随机变量A1,A2,…,An。使用极大似然估计即求取以下经验风险函数的极值点
L(P)=−∑(logP(Ai)) = −klogθ−(n−k)log(1−θ)
求导：
θ=k/n

贝叶斯估计

贝叶斯统计学派：通过观察到的现象对概率分布中的主观认定不断进行修正。

总
k近邻法是基本且简单的分类与回归方法，对给定的训练实例点和输入实例点，首先确定输入实例点的k个最近邻训练实例点，然后利用这k个训练实例点的类的多数来预测输入实例点的类。
三要素：距离度量方法，k值选择，分类决策规则。距离一遍用欧式距离，以及更一般的Lp距离。k值小，模型复杂，k值大，模型简单。k值的选择反应近似误差和估计误差的权衡，通常由交叉验证来选择最优的k。分类决策是多数表决，对应于经验风险最小化。
利用kd树可以方便搜索最近k个点（二叉树的一种）。

3.3
kd树的k邻搜索算法
输入：已构造的kd树：目标点x;
输出：x的k邻。

第一种：
（1）在kd树中找出包含目标点x的叶结点：从根结点出发，递归的向下访问kd树。若目标点x当前维的坐标小于切分点的坐标，则移动到左子结点，否则移动到右子节点，直到子节点为叶结点为止。
（2）以此叶节点为“当前最近点”。
（3）递归的向上回退，在每个结点进行以下操作：
（a） 如果该结点保存的实例点比当前最近点距离目标点更近，则以该实例点为“当前最近点”。
（b）当前最近点一定存在于该结点一个子结点对应的区域。检查该子结点的另一子结点对应的区域是否有更近的点。 具体的，检查另一子结点对应的区域是否与以目标点为球心，以目标点与“当前最近点”间的距离为半径的超球体相交。 如果相交，可能在另一个子结点对应的区域内存在距离目标点更近的点，移动到另一个子结点，接着，递归地进行最近邻搜索。
如果不相交，向上回退。
（4）保存最近点，搜索第2-k近点，重复1-3步骤，忽视已经保存过的节点.

第二种：
（1）在kd树中找出包含目标点x的叶结点：从根结点出发，递归的向下访问kd树。若目标点x当前维的坐标小于切分点的坐标，则移动到左子结点，否则移动到右子节点，直到子节点为叶结点为止。
（2）以此叶节点为“当前最近点”，保存到长度为k的数组中。
（3）递归的向上回退，在每个结点进行以下操作：
（a） 如果该结点保存的实例点比当前最近点距离目标点更近，则以该实例点为“当前最近点”，保存到长度为k的数组中。
（b）当前最近点一定存在于该结点一个子结点对应的区域。检查该子结点的另一子结点对应的区域是否有更近的点。 具体的，检查另一子结点对应的区域是否与以目标点为球心，以目标点与“当前最近点”间的距离为半径的超球体相交。 如果相交，可能在另一个子结点对应的区域内存在距离目标点更近的点，移动到另一个子结点，接着，递归地进行最近邻搜索，保存到长度为k的数组中。如果不相交，向上回退。

总结：找到最近的实例点，然后再找次近的，向外迭代即可。

2des总结

数据经过2次des加密之后，已知原文和秘文，如何破解得到两次的密钥呢？

  1. 根据已知的密钥信息，将密钥1所有可能算出，然后将des一次加密结果算出
  2. 根据已知的密钥信息，将密钥2所有可能算出，然后将des一次解密结果算出
  3. 根据结果进行碰撞检测，此处不能用普通的遍历算法，而因此利用hash表，如果没有学过hash表，可以利用python的set来进行处理，碰撞出两个数据的相同值，再反过来，根据结果，将密钥1和密钥2计算出。

总结：

先利用C进行加密和解密，然后对结果进行hash表计算，碰撞，根据结果查找key。（需要学过数据结构）
又或者导出到文本中，利用python进行set计算，然后再将结果返回查询key。（不要学过数据结构）
